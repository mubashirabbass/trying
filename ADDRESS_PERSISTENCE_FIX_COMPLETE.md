# Address & Gender Not Persisting - COMPLETE FIX ✅

## The Problem
When you saved your address (and gender) in the profile:
- ❌ The data was sent to the backend
- ❌ But it was being rejected by Zod validation
- ❌ So it was never saved to the database
- ❌ When you came back, the address was gone

## Root Cause Analysis

The issue was in **THREE places** where the Zod validation schema was incomplete:

### 1. **shared/api-zod/src/generated/api.ts** ❌ MISSING
```typescript
// BEFORE (WRONG):
export const UpdateUserBody = zod.object({
  "name": zod.string().optional(),
  "email": zod.string().optional(),
  // ... other fields ...
  "educationDocumentUrl": zod.string().optional()
  // ❌ address and gender MISSING!
})
```

### 2. **shared/api-client-react/src/generated/api.schemas.ts** ❌ MISSING
```typescript
// BEFORE (WRONG):
export interface UpdateUserBody {
  name?: string;
  email?: string;
  // ... other fields ...
  educationDocumentUrl?: string;
  // ❌ address and gender MISSING!
}
```

### 3. **shared/api-zod/src/generated/types/updateUserBody.ts** ❌ MISSING
```typescript
// BEFORE (WRONG):
export interface UpdateUserBody {
  name?: string;
  email?: string;
  // ... other fields ...
  educationDocumentUrl?: string;
  // ❌ address and gender MISSING!
}
```

## How It Works Now

### Data Flow:
```
1. User enters address in profile form
   ↓
2. Clicks "Save Changes"
   ↓
3. Frontend sends: { address: "123 Main St", gender: "male", ... }
   ↓
4. Backend receives request
   ↓
5. Zod validation checks schema
   ✅ NOW: address and gender are in schema → ACCEPTED
   ❌ BEFORE: address and gender NOT in schema → REJECTED
   ↓
6. Backend saves to database
   ↓
7. refreshUser() fetches updated data
   ↓
8. Address appears in form and everywhere
```

## Why This Happened

The Zod schemas are **generated files** (notice the comment at the top: "Generated by orval"). They were generated from the OpenAPI spec, but the spec might not have included `address` and `gender` fields in the UpdateUserBody definition.

However:
- ✅ Database schema HAS these fields
- ✅ Backend route HANDLES these fields
- ✅ Frontend SENDS these fields
- ❌ Zod validation was BLOCKING them
