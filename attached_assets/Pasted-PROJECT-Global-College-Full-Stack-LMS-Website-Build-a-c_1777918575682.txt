PROJECT: Global College — Full Stack LMS Website
Build a complete, production-ready Learning Management System for Global College,
a multi-campus vocational and technical training institute in Pakistan.

DESIGN INSPIRATION:
Study and combine the best elements of these four sites:
- https://empowerers.co — hero layout, navigation style, overall professional feel
- https://www.brighteaglets.edu.pk — course card design, popular courses section,
  Pakistani institute branding style
- https://digiskills.pk — LMS structure, video timeline system, weekly progress
  tracking, quiz + exercise flow, certificate system, how-it-works section
- https://iskills.com — WhatsApp floating button placement, social media integration,
  contact section layout, mobile responsiveness

TECH STACK:
Frontend:  React.js with Tailwind CSS, React Router v6, Axios, React Query
Backend:   Node.js with Express.js, REST API with /api/v1/ versioning
Database:  PostgreSQL with Prisma ORM (all queries through Prisma — no raw SQL)
Auth:      JWT access token (7-day) + refresh token (30-day), bcrypt cost 12
Videos:    YouTube Unlisted embeds — fully protected (AES-256 encrypted IDs)
Storage:   Cloudinary — PDFs, images, receipts, CNIC scans, certificates, photos
Email:     Nodemailer + SendGrid SMTP
PDF:       Puppeteer — certificate generation + report exports
Exports:   exceljs — Excel report exports
Push:      Web Push API with VAPID keys — in-app + browser notifications
Security:  helmet, cors, express-rate-limit, xss-clean, express-validator

VIDEO PROTECTION SYSTEM (YouTube Unlisted — Maximum Security):

Server-side (never expose YouTube ID to browser):
1. Store all YouTube video IDs encrypted using AES-256-CBC with IV in the database
   Field name: encryptedYoutubeId — never select this field in public API responses
2. Create endpoint: GET /api/v1/student/lessons/:lessonId/stream-token
   - Authenticate the student's JWT
   - Verify the student is enrolled in the course this lesson belongs to
   - Decrypt the YouTube video ID server-side
   - Generate a signed short-lived token (JWT, 30 min expiry) containing the video ID
   - Return only the token — never the raw YouTube ID
   - Log the access: userId, lessonId, ipAddress, userAgent, timestamp
3. Frontend calls this endpoint first, receives the token
4. Frontend sends the token to: GET /api/v1/student/lessons/:lessonId/embed
   - Server validates the token
   - Returns the constructed embed URL using youtube-nocookie.com
   - URL parameters: modestbranding=1, rel=0, controls=1, iv_load_policy=3,
     playsinline=1, enablejsapi=1
5. Frontend renders the embed URL inside a sandboxed iframe:
   - sandbox="allow-scripts allow-same-origin allow-presentation"
   - referrerpolicy="strict-origin"
   - Place a transparent overlay div absolutely positioned over the iframe
   - overlay CSS: position:absolute, top:0, left:0, width:100%, height:100%,
     z-index:10, background:transparent
   - overlay prevents right-click: onContextMenu={(e) => e.preventDefault()}
   - overlay shows "Content Protected — Global College" on right-click attempt
   - Do not allow the iframe src to be inspectable in browser devtools
     by setting the src dynamically after component mounts using a ref

VIDEO COMPLETION TRACKING (100% accurate, fraud-resistant):
1. Load YouTube IFrame API (youtube.com/iframe_api) dynamically
2. Use onReady callback to get player reference
3. Use onStateChange to detect PLAYING (1), PAUSED (2), ENDED (0)
4. When PLAYING: start an interval every 5 seconds:
   - currentTime = player.getCurrentTime()
   - duration = player.getDuration()
   - watchedPercent = Math.round((currentTime / duration) * 100)
   - Call PATCH /api/v1/student/lessons/:id/progress with watchedPercent
     and currentPositionSeconds
5. When PAUSED or ENDED: clear interval, immediately save progress
6. Backend upserts LessonCompletion record (@@unique on userId+lessonId)
7. When watchedPercent >= completionThreshold (from Settings, default 80):
   - Mark isCompleted = true, set completedAt = now()
   - Call checkCourseCompletion(userId, courseId) service
8. On lesson load: fetch lastPositionSeconds from backend,
   call player.seekTo(lastPositionSeconds) when player is ready
9. Fraud prevention: track sessionWatchSeconds in memory.
   If student jumps seek bar to end without watching, require
   actual elapsed real time >= (duration * threshold / 100) seconds
   before marking complete
10. Sidebar updates lesson icon to green checkmark (✓) immediately
    after successful completion API response

SECURITY IMPLEMENTATION:
- helmet({ contentSecurityPolicy: { directives: {
    defaultSrc: ["'self'"],
    scriptSrc: ["'self'", "https://www.youtube.com", "'unsafe-inline'"],
    frameSrc: ["https://www.youtube-nocookie.com"],
    connectSrc: ["'self'"],
    imgSrc: ["'self'", "https://res.cloudinary.com", "data:"],
  }}})
- cors({ origin: process.env.FRONTEND_URL, credentials: true })
- express-rate-limit: { windowMs: 15*60*1000, max: 5 } on /auth/login
- express-rate-limit: { windowMs: 60*60*1000, max: 10 } on /auth/register
- bcrypt.hash(password, 12)
- Account lock: after 5 failed logins → set lockedUntil = now + 15 minutes
- All encrypted YouTube IDs: use crypto.createCipheriv('aes-256-cbc', key, iv)
- NEVER select encryptedYoutubeId field in any Prisma query sent to frontend
- All upload endpoints: validate MIME type using file-type library before Cloudinary
- All Prisma queries use .select() to exclude sensitive fields
- Video access tokens: jwt.sign({ lessonId, userId }, secret, { expiresIn: '30m' })
- Environment validation on startup: if any required env var is missing, throw
  error and refuse to start

CODE QUALITY STANDARDS:
- All async route handlers wrapped in catchAsync(fn) utility
- All errors thrown as new AppError(message, statusCode, errorCode)
- Global error handler in middleware/errorHandler.js converts AppError to JSON response
- Standard response format always: { success, data, message } or { success, error, code }
- All business logic in /services — controllers only call services and send responses
- Prisma transactions ($transaction) for any multi-table write operations
- Winston logger with levels: error, warn, info, debug — no console.log in production
- All list endpoints paginated: default limit=20, max limit=100, returns
  { data, total, page, totalPages }
- All env vars validated at startup using a validateEnv() function
- Database indexes on all foreign keys and frequently filtered fields
- Use Prisma's select to explicitly whitelist returned fields — never return all fields
